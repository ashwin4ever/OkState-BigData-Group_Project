Intro:

  • Pattern matching is the act of checking a given sequence of tokens for the presence of the constituents of some pattern.
  • In contrast to pattern recognition, the match usually has to be exact. The patterns generally have the form of either sequences or tree structures.
  • Uses of pattern matching include outputting the locations (if any) of a pattern within a token sequence, to output some component of the matched pattern, and to substitute the matching pattern with some other token sequence (i.e., search and replace).
  • Sequence patterns (e.g., a text string) are often described using regular expressions and matched using techniques such as backtracking.

    • Hopfield Networks:

  • A Hopfield network is a form of recurrent artificial neural network popularized by John Hopfield in 1982, but described earlier by Little in 1974.
  • Hopfield networks also provide a model for understanding human memory.
  • We are using an Association and Recall model (ARM) to understand the pattern. This type of functionality is commonly seen in Hopfield Nets. It is an Artificial Neural Network (ANN) which works similar to human brain in terms of auto associative memory capabilities.
  • Data is converted into binary vectors and fed into Hopfield Nets which then learns the patterns by creating Weighted Matrices. These matrices represent the memory of the network and can be queried to recover patterns. Similarity measures like cosine similarity score is used to evaluate the similarity of the returned pattern to the pattern stored in the memory of the network.

    Proposed Mapper Architecture:


      Insight into mapper Architecture:
    • Raw logs is cleaned/processed to omit unnecessary/absent fields to reduce overhead
    • Represent the unique folders accessed by user as binary form,Since the Resources accessed is a multi level hierarchy, for simplicity sake consider up to two level and represent these multiple resources as a binary representations to form combined binary pattern
    • Processed data is then denoted in a vector space using pandas library
    • Each key will have a corresponding binary representation of the resources accessed per day/per user
    • Train the Algorithm using the matrix constructed with binary patters(generated for every user)
    • The class properties get-Weight(user defined in core library of neupy algorithm) will return the weigh matrix for the current key
    • The weight matrix then summed with on-coming key's matrix(x) such that this process is repeated for every key in the processed dataset
